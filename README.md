#  VamSirLike_Project
 ## 프로젝트 개요 
본 프로젝트는 뱀파이어 서바이벌라이크 2d 탑뷰 게임으로 데이터를 다루는 방식과 struct를 활용한 가비지 방지, 프레임 워크 도입 경험을 목표로 진행하였습니다.


##  개발 환경

>  개발 인원: 1인  
개발 기간: 2026.03.15~2026.03.02  
형상 관리: Github Desktop, GitBash ClI    
게임 엔진: Unity 6000.0.56f1   
IDE: Rider 2025.2.1

## 핵심 구현 내용
###  1. **VContainer, UniTask 활용**
  * 강한 결합과 추적이 힘든 싱글톤을 사용하지 않고 의존성주입을 위한 VContainer 도입
  * struct를 활용하여 가비지 없이 비동기 로직을 사용할수 있는  UniTask 도입
###  2.   **데이터 임포트 툴 구현**
  *  협업을 위해 기획자들이 편하게 데이터 수정을 적용할수있는 커스텀 툴 구현 
###  3. **Github Actions CI 구축**  
  *  새로운 코드가 병합될 때마다 GitHub 서버에서 자동으로 유니티 안드로이드 빌드를 수행하는 CI 파이프라인 구축
### 4. **셰이더 그래프 활용**
 * CPU에서 색상을 계산하지 않고 GPU의 프래그먼트 연산을 활용하여 CPU 부담 갑소
 * 코드내부에서 프로퍼티 블록을 사용해 material 복제로 배칭이 깨지는 문제 방
##  이슈 해결사항  
###  1.  **텍스처 압축 문제**
  *  **문제:** 빌드시 색이 번지는 문제
  *  **원인:** 안드로이드의 텍스처 압축인 ASTC가 픽셀 그래픽 일때는 문제를 일으킴
  *  **해결:** RGBA 32 bit 포맷으로 변경하여 해
###  2.  **네이티브 메모리 누수 문제** 
  *  **문제:** Addressable로 이미지를 불러왔다 제거할때 메모리누수 발생 
  *  **원인:** C# 영역에서의 참조만 사라지고 네이티브 영역의 실제 데이터가 메모리를 차지함
  *  **해결:** 이미지를 불러올때 AsyncOpperationHandle 변수에 기록 해놓았다가 사용이 끝나면 Realease 해줌

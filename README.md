#  VamSirLike_Project
 ## 프로젝트 개요 
본 프로젝트는 뱀파이어 서바이벌장르 게임으로 **오브젝트 풀**을 이용한 대규모 적 웨이브와 **상속**을 이용한 다양한 몬스터 바리에이션 및 상호작용 시스템, 
 **전략 패턴**을 이용한 무기교체 시스템을 구현하는 것을 목표로 진행하였습니다.


##  개발 환경

>  개발 인원: 1인  
개발 기간: 2026.03.15~2026.03.02  
형상 관리: Github Desktop, GitBash ClI    
게임 엔진: Unity 6000.0.56f1   
IDE: Rider 2025.2.1

## 핵심 구현 내용
###  1. **전략패턴을 이용한 무기 시스템**
  * 추상 클래스 WeaponBase를 상속하여 같은 타입으로 다룰수 있는 다양한 무기 구현
  * 플레이어의 1,2,3 입력에 따라 무기를 비활성화 / 활성화 하는 무기 교체시스템 구현
###  2.   **인터페이스를 활용한 상호작용 시스템**
  *  인터페이스 Iinteractable을 상속하여 같은 타입으로 플레이어와 상호작용할 수 있는 객체 구현 
  *  플레이어가 E 키를 입력하면 객체에 따라 다른 상호작용 로직 실행
###  3. **오브젝트 풀을 활용한 대규모 웨이브**  
  *  오브젝트 풀을 이용하여 Istantiate를 할때 일어나는 과부하 타이밍을 게임 시작으로 제어하여 쾌적화
  *  선입선출의 자료구조 Queue를 통해 오브젝트를 재사용하여 Destroy로 인한 가비지 발생 방지
### 4. **상태패턴을 이용한 보스 몬스터**
 * 기존 코드 수정없이 클래스를 추가하여 확장할수 있는 상태패턴 기반 FSM 구현
 * 상태 내부에서 조건에 따라 전이
##  이슈 해결사항  
###  1.  **몬스터 데미지 중복 문제**
  *  **문제:** 몬스터에게 데미지가 두번 들어가는 문제
  *  **원인:** 적 공격시 카메라를 급격하게 움직여 콜라이더에 두번 닿으면 한번의 공격에 데미지가 두번 들어감
  *  **해결:** 해시코드를 중복 검사로 O(1) 시간 복잡도를 가지는 HashSet을 이용하여 데미지를 입은 대상 필터링
###  2.  **GPU 병목 현상 해결** 
  *  **문제:** 런타임 중 프레임이 15 미만으로 저하되어 게임 진행이 불가능한 현상 발생
  *  **원인:** 유니티 프로파일러를 통해 GPU 병목 현상이 원인임을 파악
  *  **해결:** Occlusion Culling을 사용하여 보이지 않는 메쉬 렌더링을 막아 연산량을 줄임
## 회고 
###  1.  **정답보다는 선택과 집중** 
 * 최적화를 위해 오브젝트 풀을 도입하면서 자료구조 Queue를 사용하여 중간 데이터 접근 기능을 포기하는 대신 입출력 속도라는 이점을 얻었습니다. 
또한 Instantiate를 게임 시작 부분으로 몰아주어 부하가 걸리는 타이밍을 제어 해주어 런타임 중 프레임 드랍을 방지하였습니다.
위에 겪은 상황처럼 코드 최적화는 단순한 과부하 제거가 아닌 아니라 현재 시스템의 목표를 위해 무엇을 포기하고 무엇에 집중할 것 인가를 선택하는 과정임을 배웠습니다.
### 2. **상속의 개념에 대한 이해**
 * 프로젝트 초기 무기 교체 시스템을 구현하며 Player 클래스가 Sword 같은 구체적인 무기 클래스를 직접 참조하도록 설계했습니다. 
이로 인해 무기가 추가될 때마다 플레이어 코드를 수정해야 하는 높은 의존성 문제가 발생했습니다.
해당 문제를 해결하기 위해 WeaponBase 추상 클래스를 도입하여 전략 패턴 구조로 리팩토링했습니다. 
플레이어는 이제 구체적인 무기가 아닌 부모 타입에 의존하게 되어, 코드 수정 없이도 새로운 무기를 손쉽게 확장할 수 있는 구조가 되었습니다.
과거에는 상속을 단순히 코드 중복을 줄이는 용도로만 생각했습니다. 하지만 이번 프로젝트를 통해 다형성을 활용해 결합도를 낮추고 시스템을 유연하게 설계하는 것이 상속의 핵심 용도임을 깨달았습니다.
